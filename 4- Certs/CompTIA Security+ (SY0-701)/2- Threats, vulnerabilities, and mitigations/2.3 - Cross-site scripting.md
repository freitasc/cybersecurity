## What is it?

- One of the most common vulnerabilities in web based apps
- Takes advantage of the trust a user has on a web site
- It is the act of injecting script into a website, that runs it along its common resources
- Usually javascript based
## Additional Information

- Non-persistent
	- Search queries
	- Embedded into the URL
	- Can use credentials, cookies and IDs as part of the attack
	- Can be made into a URL that carries the attack with it
	- The URL can be sent in a email that compromises a target
- Persistent 
	- Script is saved into a database
	- When the engine brings the database result into the web screen, it runs it
	- Doesn't need another vector like an email, making it way more powerful
	- Can be used on a social media or communication platform to attack multiple users simultaneously
		- Think of twitter or a forum
	- Usually it isn't target at a single user
