## What is it?

- Leaving a door open for attackers 
- Very very common on the cloud landscape
## Additional Information

- Unsecured admin accounts
	- Linux root user using default or easy to break credentials
	- Disabling the "run as admin" security on windows
	- The best practice is disabling direct login to the root/admin account
- Insecure protocols
	- Some protocols aren't encrypted or are legacy
		- Telnet, FTP, SMTP, IMAP
	- You can verify with packet capture
		- Wireshark
	- Use the encrypted versions
		- SSH, SFTP, IMAPS
-  Default settings
	- You must reconfigure the password of all the new devices being plugged into the network
	- A common vulnerabilities in routers, specially home routers
	- admin admin
	- Mirai botnet is an example of a botnet that takes advantage of this
		- It has a big list of common default login and passwords
		- It scans the net to find misconfigured devices
		- Hacks the device and adds it to the botnet
		- Cameras, routers, doorbells, smart TVs etc
- Open ports and services
	- Services open ports naturally, so it needs to be kept in check
	- Open ports must be addressed by a firewall
	- Firewall rules sometimes are complex, great for attackers
