## Additional Information

- DNS poisoning
	- Modify the DNS server
		- Requires some expertise
	- Modify the client host file
		- The host file takes precedent over the DNS queries
	- Send a fake response to a valid DNS request
		- Requires a redirection of the original request or the resulting response
		- Real-time redirection
		- This is a on-path attack
- Domain Hijacking
	- Get access to the domain registration and you have control where the traffic flows
	- You don't need to touch the actual servers
	- Determines the DNS names and DNS IP address
	- Many ways to get into the account
		- Brute Force
		- Social engineer
		- Break into the email that owns the account
		- etc
- URL Hijacking
	- Redirect users to other site
	- Make money by redirecting to a advertise
	- Or redirect to a competitor (legal issues)
	- Transform it into a phishing site so the attacker can target the users
	- Typosquatting is an example of URL Hijacking
		- It is the act of taking advantage of poor spelling to make the user do a mistake
		- google.com vs googlle.com
		- useful for phishing
