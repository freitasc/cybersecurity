## What is it?

- The way data is, is going to be or has been
## Additional Information

- Data at rest
	- Data is on a storage device
		- HD, SSD, flash drive etc
	- Encrypt the data
		- Whole disk encryption
		- Database encryption
		- File or folder level encryption
	- Apply permissions
		- Access control lists
		- Only authorized users can access the data
- Data in transit
	- AKA data in motion
	- Data transmitted over the network
	- Not much protection as it travels
	- Network-based protection (layer 4 and 3)
		- Firewalls, IPS
	- Provide transport encryption
		- TLS and IPSEC
- Data in use
	- Data is actively processing in memory
		- RAM, CPU, cache
	- Data is almost always decrypted
	- The attackers can pick the decrypted information of a RAM
		- Very attractive
- Data sovereignty
	- Data that resides in a contry is subject to the laws of that country
	- Laws may prohibit where data is stored
		- GDPR (General Data Protection Regulation)
			- Data collected on EU citizen must be stored in the EU
		- LGPD (Lei Geral de Proteção de Dados)
			- Brazilian law
- Geolocation
	- Location details
	- Many ways to determine location
		- 802.11, mobile providers, GPS
	- Can be used to manage data access
		- Prevent access from other countries
		- Netflix blocking you from using VPN
		- Brazilian government doesn't let you use gov.br outside of brazil
	- Limit administrative tasks unless secure area is used
		- Permit enhanced access when inside the building
