## What is it?

- Test yourself before an actual event
## Additional Information

- Use well-defined rules of engagement
	- DO NOT TOUCH THE PRODUCTION SYSTEMS DUMMY
- Very specific scenario
	- Limited time to run the event
- Evaluate response
	- Document and discuss
- Tabletop exercises
	- Performing a full-scale disaster drill can be costly
	- Many of the logistics can be determined through analysis
	- Get key players together for the tabletop exercise
- Fail over
	- A failure is often inevitable
		- It's "when", not "if"
	- We may be able to keep running
		- Plan for the worst
	- Create a redundant infrastructure
		- Multiple routers, firewalls, switches
	- If one stops working, fail over to the operational unit
- Simulation
	- Test with a simulated event
		- Phishing attack, password requests, data breaches
	- Going phishing
		- Create a phishing email
		- Send to your actual user community or employees
		- See who bites
		- Test internal security
			- Did the phishing get past the filters?
		- Test the users
			- Who clicked?
			- Additional training may be required (or firing people)
- Parallel processing
	- Split a process through multiple CPUs
		- A single computer with multiple CPU cores
		- or multiple physical CPUs
		- or multiple computers
	- Improved performance
		- Split complex transactions across multiple processors
