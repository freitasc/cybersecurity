## Additional Information


- Rules of engagement
	- A document describing how and what you can do and what is off limits
		- IP address ranges
		- Emergency contacts
		- How to handle sensitive info
		- In-scope and out-of-scope devices or apps
	- The type of test and schedule
		- On-site physical breach, internal test, external test
		- What time it's supposed to happen
- Exploiting
	- Try to break in
		- Be careful, this can break shit
		- Buffer overflows can cause instability and DDoS might break deployment
	- The objective is trying to gain privilege escalation
	- You will need to try many many different vulnerabilities and exploits
	- You will only be sure you're vulnerable if you can prove its possible to be bypassed
- The process
	- Exploit and get in
	- Lat and Vert movement across different systems and users
	- Gain persistence to come back later
	- Gain a pivot into something that should not be normally possible
- Responsible disclosure program
	- Even if a vulnerability is discovered, sometimes it takes time until you can fix it
	- Bug bounties is a way to get paid by finding a vulnerability
		- Usually only big companies do this
		- Needs to be very well documented
