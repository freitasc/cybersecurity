## What is it?

- Ensures the right people access the right resources, for the right reasons, and at the right time.
## Additional Information

- Identify life cycle management
	- Every entity gets a digital identity
- Access control
	- An entity only gets access to what they need
- Authentication and authorization
	- Entities must prove they are who they claim to be
- Identify governance
	- Track an entity's resource access
	- May be a regulatory requirement

- Provisioning/de-provisioning user accounts
	- The user account creation and removal process
	- Provisioning and de-provisioning occurs for certain events
		- Hiring, transfers, promotions, job separation
	- Account details
		- Name, attributes, group, permissions
	- An important part of the IAM process
- Permissions assignments
	- Each entity gets limited permissions
		- Just enough to do their job
		- Group assignments are common
	- Storage and files can be private to that user
	- No privileged access to the operating system
- Identity proofing
	- I could be anyone
	- Resolution
	- Validation
	- Verification / Attestation
- SSO (Single sign-on)
	- Provide credentials one time
	- Usually limited by time
	- The underlying authentication infrastructure must support SSO
- LDAP (lightweight directory access protocol)
	- Protocol for reading and wrighting directories over an IP network
	- X.500 specifications was written by the International Telecommunications Union (ITU)
	- DAP ran on the OSI protocol stack
	- LDAP is the protocol used to query and update an X.500 directory
		- Used in windows active directory, apple opendirectory, novell eDiretory etc
- SAML (Security assertion markup language)
	- Open standard for authentication and authorization
	- Not originally designed for mobile apps
	- ![[Pasted image 20260209132751.png]]
- OAuth
	- Authorization framework
		- Determines what resources a user will be able to access
	- Created by twitter, google and many other megacorps
	- Not an authentication protocol, but an authorization one
		- OpenID Connect handles the single sign-on authentication
		- OAuth provides authorization between apps
- Federation
	- Provides network access to others
	- Third parties can establish a federated network
	- The third-parties mus establish a trust relationship
- Interoperability
	- Many different ways to communicate with an authentication server
	- Often determined by what is at hand
	- A new app uses OAuth
	- The interoperability is dependent on the environment
