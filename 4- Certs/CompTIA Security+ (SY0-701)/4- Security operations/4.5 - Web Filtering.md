## What is it?

- Control traffic based on the data within the content
## Additional Information

- Corporate control of outbound and inbound data
- Control of inappropriate content
- Protection against evil
- URL scanning
	- Allow or restrict based on Uniform Resource Locator
	- Managed by category
	- Can have limited control
	- URLs aren't the only way to surf
	- Often integrated into a NFGW
	- Block rules
		- Based on specific URL
		- Category of site
		- Different dispositions depending on the organization
		- Reputation
			- Filter URLs based on perceived risk
			- Trustworthy, low risk, medium risk, suspicious, high risk
			- Automated reputation
			- Manual reputation
- Agent Based
	- Install client software on the user's device
	- Users can be located anywhere
	- Updates must be distributed to all agents
- Proxies
	- Sit between the users and the external network
	- Receives the users requests and sends the request on their behalf
	- Apps may need to know how to use the proxy
	- Some proxies are invisible (transparent)
	- Forward proxy
		- Centralized internal proxy
- DNS filtering
	- Before connecting to a website, get the IP address
	- DNS is updated with real-time threat intelligence
	- Harmful sites are not resolved by the DNS
		- No IP address, no connection
		- This works for any DNS lookup
