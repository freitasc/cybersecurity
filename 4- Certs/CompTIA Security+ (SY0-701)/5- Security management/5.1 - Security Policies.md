## What is it?

- Security rules defined by the organization

## Additional Information

- CIA (Confidentiality, integrity and availability)
- Detailed security goals
- Security policies answer the "what" and "why"

- Information security policies
	- A bit list of all security-related policies
	- Compliance requirements
	- Detailed security procedures
	- A list of roles and responsibilities
	- This is just words
- AUP (Acceptable use policies)
	- What is acceptable use of company assets?
	- Covers many topics
		- Internet use, telephones, computers, mobile devices, browsers etc
	- Used by an organization to limit legal liability
		- If someone is dismissed, these are well-documented reasons why
- Business continuity
	- Not everything goes according to plan
	- We rely on our computer systems
	- There needs to be an alternative
		- Manual transactions
		- Paper receipts
		- Phone calls for transactions approvals
	- These must be documented and tested before a problem occurs
- Disaster recovery plan
	- If a disaster happens, IT should be ready
	- Disasters are many and varied
	- A comprehensive plan
- Incident response roles
	- Incident response team
		- Specialized group, trained and tested
		- IT security management
		- Compliance officers
		- Technical staff
		- User community
- NIST SP800-61
	- https://csrc.nist.gov/pubs/sp/800/61/r3/final
- Software developemnt lifecycle (SDLC)
	- Many ways to get from idea to app
	- Many moving parts
	- Customer requirements
	- Keep the process on schedule
	- Stay on budget
	- There's no "best way"
- Change management
